# Story 1.3: Portfólio e Página de Serviços

**Epic:** 1 - Site Institucional
**ID:** STORY-1.3
**Status:** Done
**Data Criação:** 26/02/2026

---

## 📋 Descrição

Desenvolver seção de portfólio com galeria de projetos/serviços realizados, permitindo filtros por categoria e exibição de detalhes completos de cada case study com resultados e tecnologias utilizadas.

---

## ✅ Critérios de Aceitação

**Given** um usuário visitando a página de portfólio
**When** interage com a galeria
**Then**:
- [x] Galeria exibe mínimo 6 projetos em grid responsivo
- [x] Filtros por categoria funcionam (filtro "Todos" como padrão)
- [x] Click em projeto abre modal ou página de detalhes
- [x] Detalhes do projeto incluem: nome, descrição, tecnologias, resultados, imagens
- [x] Página lista 3+ categorias (ex: Web, Mobile, Design)
- [x] Search por nome/categoria busca projetos (opcional)
- [x] Breadcrumb navigation presente
- [x] CTA "Solicitar proposta" presente em cada projeto
- [ ] Performance: imagens otimizadas, scroll suave
- [x] Mobile-first: galeria se adapta perfeitamente

---

## 🎯 Escopo

### ✔️ Incluso (IN)

1. **Página de Portfólio (/portfolio)**
   - Hero section "Nossos Trabalhos"
   - Tabs/Botões de filtro por categoria
   - Grid de projetos com hover effect
   - Paginação ou "Load More"

2. **Página de Detalhe de Projeto (/portfolio/[slug])**
   - Imagem principal hero
   - Galeria de screenshots
   - Descrição completa
   - Tecnologias utilizadas (badges)
   - Resultados/métricas obtidas
   - Timeline do projeto (opcional)
   - Links para projeto ao vivo (se aplicável)
   - Navegação "Projeto Anterior/Próximo"

3. **Componentes Reutilizáveis**
   - `ProjectCard` — card de projeto na galeria
   - `CategoryFilter` — filtros
   - `ProjectGallery` — galeria de imagens

4. **Data Structure**
   - JSON/CMS com dados de projetos
   - Slug-based routing

### ❌ Não Incluso (OUT)

- Sistema de comentários
- Ratings/reviews
- CMS admin (v1)
- Integração com GitHub repos
- Live preview embutido
