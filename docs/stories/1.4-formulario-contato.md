# Story 1.4: Formulário de Contato e Sistema de Leads

**Epic:** 1 - Site Institucional
**ID:** STORY-1.4
**Status:** Ready
**Data Criação:** 26/02/2026

---

## 📋 Descrição

Implementar formulário de contato funcional com validação, envio de emails de confirmação e armazenamento de leads para acompanhamento posterior.

---

## ✅ Critérios de Aceitação

**Given** um visitante interessado em contatar a empresa
**When** preenche e submete o formulário
**Then**:
- [ ] Form valida campos obrigatórios (nome, email, mensagem)
- [ ] Email fornecido é validado (formato correto)
- [ ] Mensagem tem mínimo 10 caracteres
- [ ] Telefone opcional segue padrão brasileiro
- [ ] Após submit: mensagem de sucesso aparece
- [ ] Visitante recebe email de confirmação
- [ ] Empresa recebe notificação do novo lead
- [ ] Dados são armazenados em banco (ou arquivo com segurança)
- [ ] CSRF protection implementada
- [ ] Rate limiting previne abuso (max 5 submits/hora por IP)
- [ ] Mensagens de erro claras e acessíveis (a11y)

---

## 🎯 Escopo

### ✔️ Incluso (IN)

1. **Componente Form**
   - Campos: Nome, Email, Telefone (opt), Assunto, Mensagem
   - Validação real-time (feedback visual)
   - Botão submit com loading state
   - Mensagens de erro/sucesso

2. **Backend (Node.js/Express)**
   - POST `/api/contact` endpoint
   - Validação server-side
   - CSRF token generation
   - Rate limiting middleware
   - Email service integration (Nodemailer/SendGrid)

3. **Emails**
   - Email de confirmação ao visitante
   - Notificação à empresa com dados do lead
   - Templates HTML formatados

4. **Armazenamento**
   - Banco de dados local (SQLite) ou
   - Arquivo JSON criptografado
   - Campos: id, nome, email, telefone, assunto, mensagem, data, IP

5. **Segurança**
   - Input sanitization (previne XSS)
   - CSRF protection com tokens
   - Rate limiting por IP
   - Erro messages não expõem informações sensíveis

### ❌ Não Incluso (OUT)

- CRM integration (v1)
- Chatbot
- Notificações em tempo real
- Autenticação de usuários
- Dashboard admin complexo
- SMS notifications

## ⚠️ Riscos

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|--------|-----------|
| Email service não configurada | Média | Alto | Usar env vars, mock em dev |
| Spam/abuso do form | Alta | Médio | Rate limiting + CAPTCHA (optional) |
| Emails caem em spam | Alta | Médio | SPF/DKIM configurados |
| Dados vazados | Baixa | Crítico | Sanitização + HTTPS obrigatório |
